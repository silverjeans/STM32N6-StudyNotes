## 2. STM32N657X0H3Q MCU 핵심 사양

| 항목 | 사양 | 비교 (STM32H7) |
|------|------|----------------|
| 코어 | Cortex-M55 @ 800 MHz | M7 @ 480MHz |
| 벡터엔진 | Helium (SIMD) | 없음 |
| NPU | Neural-ART @ 1GHz, 600 GOPS | 없음 |
| 내장 RAM | 4.2 MB (연속) | 1 MB |
| ITCM/DTCM | 64KB / 128KB (ECC, 지연 0) | 64KB / 128KB |
| I/D Cache | 각 32KB | 각 16KB |
| 성능 | 3616 CoreMark | ~2400 |

### 특수 기능 (기존 STM32에 없던 것)
- **NPU (Neural-ART)**: AI 모델 추론 전용 하드웨어 가속기
- **MIPI CSI-2**: 고속 카메라 인터페이스
- **ISP**: 하드웨어 이미지 신호 처리 (자동 노출, 화밸)
- **H.264 인코더**: 하드웨어 비디오 압축
- **NeoChrom GPU**: 2D 그래픽 가속

## 3. NUCLEO-N657X0-Q 보드 특징

- ST-LINK 디버거 내장 (별도 장비 불필요)
- USB Type-C 연결 (전원 + 디버깅 + 시리얼)
- Arduino Uno V3 커넥터 호환
- ST Morpho 커넥터 (MCU 전핀 접근)
- GPIO 158개 (125개 자유 사용 가능)
- USART/UART 10개 (콘솔: USART1, 115200 8N1)

### 주의사항
- 카메라(DCMIPP/CSI) 예제는 Discovery Kit(STM32N6570-DK)에만 있음
- Nucleo 보드에서 카메라 사용 시 직접 MIPI CSI-2 회로 구성 필요

## 4. 프로젝트 목표 - 디지털 리프랙터 동공 트래킹

### 핵심 파이프라인
```
IR 카메라 → MIPI CSI-2 → ISP(영상보정) → NPU(동공검출 AI) → 동공 좌표 출력
```

### 스터디 우선순위
1. 카메라 + 비전 파이프라인 (MIPI CSI-2, ISP, DCMIPP)
2. AI/NPU 활용 (ST Edge AI Suite, 모델 변환/배포)
3. 실시간 처리 (ITCM/DTCM, DMA, 타이머/인터럽트)
4. 통신 (UART/SPI로 좌표 전송)

## 5. x-cube-n6-ai-h264-usb-uvc 예제 분석

### 구조 (동공 트래킹과 거의 동일한 구조)
- **3개 FreeRTOS 스레드** 구조
  - NN Thread (우선순위 +1): NPU 추론
  - DP Thread (우선순위 -2): 후처리 + 출력
  - ISP Thread (우선순위 +2): 이미지 보정

### 핵심 파일
| 파일 | 역할 |
|------|------|
| `Src/app.c` | 메인 로직 - 3개 스레드, 버퍼 큐, 전체 흐름 |
| `Src/app_cam.c` | 카메라 초기화, DCMIPP 듀얼 파이프 설정 |
| `Src/main.c` | 시스템 초기화, 클럭 800MHz/NPU 1GHz |
| `Src/utils.c` | NPU 비동기 추론 실행 함수 |
| `Model/` | AI 모델 파일 (TinyYOLOv2 224x224) |

### app.c 핵심 흐름
```
app_run() [755줄]
├── CAM_Init() → 카메라 초기화
├── ENC_Init() → H264 인코더
├── UVCL_Init() → USB 비디오
├── CAM_DisplayPipe_Start() → 캡처 시작
└── 3개 스레드 생성
```

### 더블 버퍼 큐 (bqueue) 패턴
- `nn_input_queue`: 카메라 프레임 → NN 스레드 전달
- `nn_output_queue`: NN 결과 → DP 스레드 전달
- 버퍼 2개씩 → 처리 중에도 다음 데이터 준비 (파이프라이닝)

### nn_thread_fct() [369줄] - NPU 추론 루프
```c
stai_runtime_init();           // NPU 런타임 초기화
stai_network_init(network_ctx); // 모델 로드

while (1) {
    capture = bqueue_get_ready(&nn_input_queue);   // 카메라 프레임 받기
    output = bqueue_get_free(&nn_output_queue, 1);  // 출력 버퍼 확보
    stai_network_set_inputs(ctx, inputs, 1);        // 입력 설정
    stai_network_set_outputs(ctx, outputs, 1);      // 출력 설정
    Run_Inference(ctx);                              // NPU 추론 (비동기)
    bqueue_put_ready(&nn_output_queue);             // 결과 전달
}
```

### dp_thread_fct() [664줄] - 후처리 루프
```c
while (1) {
    output = bqueue_get_ready(&nn_output_queue);   // NPU 결과 받기
    app_postprocess_run(pp_input, 1, &pp_out, &pp_params); // YOLO 후처리
    display_new_frame(&pp_out);                     // 화면 출력
}
```

### 카메라 DCMIPP 듀얼 파이프
- PIPE1: 1280x720 ARGB8888 → 디스플레이용 (capture_buffer[3])
- PIPE2: 224x224 RGB888 → NN 입력용 (nn_input_buffers[2])
- 프레임 이벤트 콜백으로 자동 버퍼 순환

### 메모리 배치
| 버퍼 | 위치 | 크기 |
|------|------|------|
| capture_buffer[3] | PSRAM (외부) | 3.5MB |
| nn_input_buffers[2] | PSRAM (외부) | 150KB |
| nn_output_buffers[2] | 내부 RAM | 5.8KB |
| network_ctx | 내부 RAM | 모델 컨텍스트 |
| 모델 가중치 | xSPI2 Flash | 10.5MB @ 0x70380000 |

### 동공 트래킹으로 변경 시 수정 포인트
| 현재 | 변경 |
|------|------|
| TinyYOLOv2 (사람 검출) | 동공 검출 모델 (MediaPipe Iris 등) |
| 224x224 입력 | 64x64 (모델에 따라) |
| YOLO 후처리 | 동공 좌표 후처리 |
| H264 + USB 출력 | UART로 좌표 전송 |

## 6. 동공 검출 AI 모델 조사 결과

### 후보 모델 비교
| 모델 | 입력 | 출력 | 특징 |
|------|------|------|------|
| **MediaPipe Iris** (추천) | 64x64 RGB | 동공 중심 + 홍채 4점 + 눈 71점 | 경량, TFLite 제공, Google 검증 |
| Pupil-locator | 눈 이미지 | 동공 (x,y) 좌표 | YOLO 기반, CPU 34fps |
| DeepVOG | 눈 이미지 | 동공 마스크 + 시선 | FCN, 정확하지만 무거움 |

### MediaPipe Iris 선정 이유
- 입력 64x64 → NPU 부담 최소
- 동공 중심 좌표 직접 출력 → 후처리 간단
- TFLite 포맷 → ST Edge AI Suite로 NPU 변환 가능
- 리프랙터 특성상 얼굴 검출 불필요 (눈만 촬영)

### 리프랙터 적용 시 장점
- 환자 얼굴이 기기에 고정됨 → 얼굴 검출 단계 생략
- IR 카메라가 눈만 촬영 → 바로 동공 검출 모델 적용
- 조명 통제 가능 → 정확도 높음
